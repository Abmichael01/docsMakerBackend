SVG PARSER CONVENTIONS
======================

This document explains how to name SVG element IDs to create form fields automatically.

BASIC FIELD TYPES
=================

1. TEXT FIELD
   ID: Company_Name.text
   Meaning: Creates a text input field with the label "Company Name"

2. TEXTAREA FIELD  
   ID: Description.textarea
   Meaning: Creates a large text area with the label "Description"

3. DROPDOWN FIELD
   ID: Country.select_OPTION1
   ID: Country.select_OPTION2  
   ID: Country.select_OPTION3
   Meaning: Creates a dropdown with options "OPTION1", "OPTION2", "OPTION3"
   Example: 
   Country.select_USA
   Country.select_Canada
   Country.select_Mexico
   Creates dropdown with USA, Canada, Mexico options

4. RANDOM CODE FIELD
   ID: Reference_Code.gen.max_8
   Meaning: Creates a field that generates random 8-character codes
   Example: Reference_Code.gen.max_10 generates 10-character codes

5. DEPENDENT FIELD
   ID: City.depends_Country
   Meaning: This field copies the same content as the "Country" field - when Country changes, this field shows the same content

SPECIAL TRACKING FIELDS
=======================

TRACKING ID FIELD
ID: Order_Number.gen.max_12.tracking_id.link_https://example.com/track
Meaning: Creates a tracking field that:
- Generates random 12-character tracking codes
- Marks it as the main tracking ID with .tracking_id
- Includes a link to https://example.com/track for tracking

TRACKING ROLE FIELDS (.track_)
⚠️ IMPORTANT: .track_ extensions MUST ALWAYS be the LAST extension in the ID chain

ID: Customer_Name.text.track_name
ID: Customer_Email.email.track_email
ID: Destination.text.track_destination
ID: Status.select_Delivered.track_status
ID: Status.select_Error.editable.track_status

Meaning: Maps fields to specific roles for tracking website display
- Common roles: track_name, track_email, track_status, track_destination, track_package, track_weight, track_error_message
- Must be the last extension (after .gen, .max_, .link_, .editable, etc.)
- For select fields with .track_, only ONE option needs the extension

EDITABLE FIELDS
ID: Status.text.editable
ID: Notes.textarea.editable
ID: Status.select_Delivered.editable.track_status

Meaning: Marks fields as editable even after document purchase
- By default, all fields become non-editable after purchase
- Must come BEFORE .track_ extension if both are used

IMPORTANT RULES
===============

1. Field names must use Title_Case (capital first letter of each word)
   Examples: Company_Name, Reference_Code, Tracking_ID

2. Tracking_ID must always be uppercase exactly as shown ("For now, will be updated later")

3. For dropdowns, create separate elements for each option
   Example: 
   Country.select_USA
   Country.select_Canada  
   Country.select_Mexico

4. For random codes, use .gen.max_N where N is the number of characters
   Example: Reference_Code.gen.max_8

5. For links, use .link_URL where URL is the full website address
   Example: Order_Number.gen.max_12.tracking_id.link_https://example.com/track

6. ⚠️ .track_ extensions MUST ALWAYS be the LAST extension
   Example: Customer_Name.text.track_name
   Example with editable: Status.text.editable.track_status

7. For editable fields, use .editable (must come before .track_ if both are used)
   Example: Notes.textarea.editable
   Example with tracking: Status.select_Delivered.editable.track_status

EXAMPLES
========

Company_Name.text
- Creates: Text field labeled "Company Name"

Reference_Code.gen.max_8
- Creates: Random 8-character code generator

Country.select_USA
Country.select_Canada
Country.select_Mexico
- Creates: Dropdown with USA, Canada, Mexico options

Order_Number.gen.max_12.tracking_id.link_https://track.example.com
- Creates: Random 12-character tracking code with link to track.example.com

City.depends_Country
- Creates: City field that copies the same content as Country field

Customer_Name.text.track_name
- Creates: Text field for customer name, mapped to "name" role for tracking sites

Customer_Email.email.track_email
- Creates: Email field for customer email, mapped to "email" role for tracking sites

Status.select_Processing
Status.select_In_Transit  
Status.select_Delivered.editable.track_status
Status.select_Error
- Creates: Status dropdown with options, editable after purchase, mapped to "status" role
- Note: .editable comes before .track_status (track_ must be last)

Error_Message.textarea.editable.track_error_message
- Creates: Editable error message field mapped to "error_message" role for tracking sites
